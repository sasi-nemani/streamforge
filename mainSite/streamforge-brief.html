The problem isn't a missing tool — it's that no one can see the estate.

Event streams are the nervous system of a modern business, but they're governed by tribal knowledge. The result is three compounding problems:

You can't see what you have

Hundreds of topics and queues across clusters and platforms. Legacy systems nobody owns. External partner feeds you can't change. No single inventory exists.

Changes break things silently

A producer renames or retypes a field; nobody is told; downstream services fail hours later. Days are lost finding what even depended on it.

Compliance can't be answered

Passport numbers, dates of birth and payment data flow through queues nobody monitors. When audit asks where PII goes, no one can say.

The cost is concrete: production incidents, weeks of engineering time on every new project, slow change, and standing audit risk — all because the contract of each stream lives only in code.

How it works

Samplenon-destructive read
→
Inferstatistical + optional LLM
→
Mapcross-topic graph + lineage
→
WatchFDR-controlled drift

Schema inference is deterministic by default — typed-frequency analysis, presence rates, pattern detection (UUID, email, timestamp), cardinality and quorum voting, with a confidence score per field (Wilson interval). An LLM is an optional enrichment for semantic naming only, can be self-hosted or disabled, and never sits on the critical path. Drift uses proven tests — binomial z, chi-squared, PSI — with Benjamini-Hochberg FDR correction to control false positives at scale.

Connector architecture & loose coupling

Non-destructive by design

Kafka: dedicated consumer group, independent offsets. SQS: peek (VisibilityTimeout 0). IBM MQ: browse mode. Your consumers and delivery are untouched.

Decoupled core

Connectors implement one read interface; inference, the cross-topic graph and drift detection never depend on the transport. New sources plug in without touching the engine.

Security & data residency

Nothing leaves your network

Runs fully deterministic with no external calls. If the optional LLM is enabled, values are pseudonymised first and it can be pointed at an on-prem model — or disabled for an air-gapped deployment.

Read-only & auditable

Never writes to brokers; messages are sampled, analysed and discarded, not stored. Every inference and drift decision is logged with its evidence for compliance review.

The differentiator: observed lineage

Most tools ask teams to declare which fields they consume — a manifest that rots and that a competitor can copy. StreamForge can observe which fields each consumer actually reads at runtime, with access counts, and compound that into a per-company access graph that gets more valuable every day it runs. So the blast radius isn't a guess: "changing gate affects the departure boards (read 8M times yesterday), the app, baggage routing and catering." It's the one asset a new competitor can't bootstrap.
